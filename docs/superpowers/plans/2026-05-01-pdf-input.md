# PDF as input format — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Accept `application/pdf` uploads on `/v1/convert/png` and `/v1/convert/markdown`; reject on `/v1/convert/pdf` with 415.

**Architecture:** Install `libreoffice-pdfimport` so PDFs flow through the existing `lok` pipeline as Draw documents. Handler layer adds one MIME-table case + one helper + one early-return. No worker code changes.

**Tech Stack:** Go 1.25, libreoffice-pdfimport (apt), gofpdf (test fixture generator), existing `lok` cgo binding.

**Branch:** `feat/pdf-input` (already created, with the spec committed).

```go
// isPDFContentType reports whether ct identifies a PDF body. Used by
// convertPDF to reject PDF→PDF on the "convert *to* PDF" route.
func isPDFContentType(ct string) bool {
	return extensionFromContentType(ct) == ".pdf"
}
```

```go
func (s *Server) convertPDF(w http.ResponseWriter, r *http.Request) {
	if isPDFContentType(r.Header.Get("Content-Type")) {
		WriteProblem(w, r.URL.Path, RequestIDFrom(r.Context()), ErrPDFNotAcceptedAsInput)
		return
	}
	s.handleConversion(w, r, worker.Job{
		Format:   worker.FormatPDF,
		Password: r.Header.Get("X-Bi-Password"),
	})
}
```

```markdown
### PDF as input

`/v1/convert/png` and `/v1/convert/markdown` accept `application/pdf`
bodies. PNG renders the requested page; markdown extracts text via the
same html → markdown pipeline used for office documents (no Marp).

`/v1/convert/pdf` rejects PDF bodies with 415 (`unsupported-media-type`)
since the route is "convert *to* PDF", not passthrough.

Scanned/image-only PDFs produce empty or near-empty markdown — there is
no OCR step. Encrypted PDFs use the standard `X-Bi-Password` header.
```
